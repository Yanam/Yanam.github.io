前言

UIBI最近在搞压力测试，数据库操作非常慢，在去掉事务后性能有很大的提升，但是后台时不时会报出"too many connections"的问题。由于很难从代码层面入手（使用的全是框架啊mybatis、dbcp什么的），所以准备监控一下数据库的运行状态。

mysql运行状况

这次主要用到了两个： 

# 查看MySQL服务器配置信息 
mysql> show variables;  

# 查看MySQL服务器运行的各种状态值 
mysql> show global status;

# 查看MySQL运行状态
mysql> show status;

# 上面的命令都是进入到mysql的命令行中运行的，这一个是直接在shell里运行的
> mysqladmin -uroot status -i 5
