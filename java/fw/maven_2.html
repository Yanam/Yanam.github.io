项目的版本

对于经常引用各种jar包的开发人员来说，version在常见不过了，但是它的格式可能很多人不清楚

<major version>.<minor version>.<incremental version>-<qualifier>
<主版本>.<次版本>.<增量版本>-<限定符>

qualifier（限定符）是指alpha、beta等用来表示里程碑。

依赖范围(denpendency scope)

  1. compile（编译范围）：Maven的默认范围
  2. provided（已提供范围）：比如你要开发一个web工程，在编译的时候需要用Servlet API的包，但是不想再打好的war包中包含这个jar。
  3. runtime（运行时范围）：在运行和测试系统的时候需要，在编译的时候不需要，比如：JDBC API 的jar，只有在运行时才需要它的实现。
  4. test（测试范围）：
  5. system（系统范围）：类似于provided，但是必须显示提供一个本地系统的jar文件路径。
