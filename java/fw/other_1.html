前言

公司有个报警系统，需要添加一个定时请求某个URL来验证服务是否可用的功能，一个很明确的定时任务应用场景，麻烦的地方是需要支持通过界面对任务进行增删改的功能。

实现

由于功能很简单，很像一个简单的线程池，就决定用Java的ScheduledThreadPoolExecutor类来实现，

添加任务

利用它的scheduleWithFixedDelay方法，因为其会返回一个“ScheduledFuture”的结果，为了后面的删除操作，添加时会把返回结果放到一个map中，key当然是用任务编号或者其他标示了。

删除任务

利用它的remove方法，参数就用之前map中存入的ScheduledFuture，这里需要强制转型为Runnable类型。
