注解(annotation)

JDK5中引入了的注解机制。虽然平时也用，但是了解不多，都是照葫芦画瓢，今天仔细学习一下。

注解使得Java源码中不但可以包含功能性的实现代码，还可以添加元数据。它与代码注释类似，不同的是它是实现程序功能的重要组成部分。

Java内置三种注解,定义在java.lang包中。

  1. @Override 表示当前方法是覆盖父类的方法。
  2. @Deprecated 表示当前元素是不推荐使用的。
  3. @SuppressWarnings 表示关闭一下不当的编译器警告信息。

开发注解


@Target{ElemnetType.TYEP}
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
public @interface Test {
	int id();
	String descrption() default "nothing"
}

@interface用来声明一个注解，其中的每个方法实际是声明了一个配置参数。方法的名词就是参数名词，返回值类型就是参数的类型，可以通过default来声明参数的默认值。

元注解 :

  1. @Target 用来声明注解可以被添加到哪些类型的元素上。
  2. @Retention 用来声明注解的保留策略，只有声明为RUNTIME时，才可以在运行时通过反射API来获取注解的信息。
  3. @Documented 将此注解包含到javadoc中
  4. @Inherited 允许子类继承父类中的注解
