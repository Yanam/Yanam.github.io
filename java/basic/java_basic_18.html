前言

最近准备好好看看JDK 的collection包，在翻源码的时候看到一个工具类java.util.Arrays，看名称就知道这是一个数组的工具类，仔细看看还是很有内容的。

Arrays

它提供了一堆静态方法：

  • sort：排序方法，实际调用的java.util.DualPivotQuicksort（它实现了双基准快速排序）
  • mergeSort：归并排序方法
  • swap：
  • legacyMergeSort：旧的归并排序，1.7以后已经改了
  • rangeCheck：
  • binarySearch：二分查找方法
  • equals
  • deepEquals
  • fill：填充方法，当需要赋默认值的时候，就别再自己写for循环了，直接用它更省事
  • copyOf
  • copyOfRange
  • asList
  • hashCode
  • toString
  • deepToString

这里面比较重要的就属sort方法了，比如下面的Collections的sort也是方法也是间接使用的Arrays的sort方法。

Collections

它提供了一系列操作collections或者返回collections的方法，包含了很多内部静态类来实现相应的功能。

主要的方法：

  • Queue<T> asLifoQueue(Deque<T> deque)：返回一个后进先出(lifo)队列
  • boolean disjoint(Collection<?> c1, Collection<?> c2)：如果两个集合没有共同的元素时返回true
  • int frequency(Collection<?> c, Object o)：返回指定对象o在指定集合c中出现的次数
  • void rotate(List<?> list, int distance)：根据指定的距离轮换指定列表的元素
  • max、min方法：返回集合中的最大值最小值，可以传入Comparator来自定义排序规则，默认是自然序列
  • void rotate(List<?> list, int distance)
  • fill(List<? super T> list, T obj)
  • void shuffle(List<?> list)/void shuffle(List<?> list, Random rnd)：使用随机源（自己指定或者默认）对指定list进行赋值
  • 一堆关于List的查找方法，因为都是List转换成对应的数组，然后使用Arrays的方法实现的，所以Arrays支持哪种排序、查找方法，它就同样支持
  • 将制定的Set、Map、List、Set等转换为不可变的、同步的、进行类型校验的等等
