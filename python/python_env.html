linux下python的安装

# 下载解压
wget http://python.org/ftp/python/2.7.2/Python-2.7.2.tgz
tar zxvf Python-2.7.2.tgz
cd Python-2.7.2
# 编译安装
./configure --prefix=/usr/local/python-2.7
make
make install
# 检查和设置环境变量
mv /usr/bin/python /usr/bin/python_bak
ln -s /usr/local/python-2.6/bin/python /usr/bin/python

setuptools

setuptools是安装pip的前置条件。 安装步骤如下：

  1. 直接下载ez_setup.py文件。
  2. 运行该文件 
    $ python ez_setup.py

pip

pip是安装virtualEnv的前置条件。 安装步骤如下：

  1. 直接下载get-pip.py文件。
  2. 运行该文件 
    $ python get-pip.py

virtualEnv

简介

virtualenv
是一个用来创建隔离的python环境的工具，它创建的每个环境有自己的安装目录，不与其他环境共享包，方便解决不同工程间的版本和依赖问题。

安装

可以使用pip工具安装
pip install virtualenv
如果想安装最新开放版
pip install https://github.com/pypa/virtualenv/tarball/develop。
也可以使用easy_install工具
sudo easy_install virtualenv

如果没有python包管理，可以直接下载virtualenv.py这个文件，然后直接运行python virtualenv.py。

使用

创建环境
# 方式一
python virtualenv.py 环境名
# 方式二
virtualenv 环境名
The --system-site-packages 选项
如果在创建环境时使用了改选项，则新创建的虚拟环境会从/usr/lib/python2.7/site-packages继承包。
virtualenv --system-site-packages 环境名

激活脚本（activate script）

执行激活脚本会将环境变量中$PATH中python的路径改为指定环境的中python的路径。命令行的提示符会有相应的显示，来标示出当前的环境名。

liunx
指定环境source 环境名/bin/activate，解除环境source 环境名/bin/deactivate

windows
指定环境\path\to\env\Scripts\activate，解除环境\path\to\env\Scripts\deactivate
